# Yushu Robot

This repository contains the local Unitree G1 29-DOF robot assets, IsaacLab
simulation examples, and Phase 1 PPO standing-training workflow.

The project is intended to stay self-contained under `yushu_robot/`. IsaacLab
itself lives outside this repository at `D:\il\IsaacLab`.

## Project Layout

| Path | Purpose |
| --- | --- |
| `yushu_robot_urdf/` | Source URDF, STL meshes, sensors, and robot-part metadata. See `yushu_robot_urdf/README.md`. |
| `model/` | IsaacLab conversion, scene, environment, sensor, terrain, training, and playback code. See `model/README.md`. |
| `model/generatedUSD/` | Generated robot USD output used by later IsaacLab steps. |
| `logs/rsl_rl/g1_stand/` | Local RSL-RL training logs and checkpoints. This directory is generated by training. |
| `PROJECT_PLAN.md` | Project planning and handoff notes. |
| `spec.md` | High-level training scenario notes. |
| `docs/superpowers/plans/` | Local implementation plans written during development. |

## Main Workflows

### Build the G1 USD

```powershell
D:\il\env\Scripts\python.exe -B model\build_robot_model\build_g1_from_urdf.py --headless
```

This validates the local URDF and meshes, then writes:

```text
model/generatedUSD/g1_29dof_mode_16.usd
```

### Smoke-Test the Robot and Environments

```powershell
D:\il\env\Scripts\python.exe -B model\step1_load_articulation\test_step1_load_articulation.py -v
D:\il\env\Scripts\python.exe -B model\step2_create_scene\test_step2_create_scene.py -v
D:\il\env\Scripts\python.exe -B model\step3_robot_cfg\test_g1_local_cfg.py -v

D:\il\env\Scripts\python.exe -B model\step5_sensors\test_sensors_cfg.py -v
```

### Test Terrains

```powershell
D:\il\env\Scripts\python.exe -B model\step6_train\terrains\test_terrains.py --terrain T0 --headless --max_steps 2
```

Available terrain keys are `T0` through `T5`.

### Train Phase 1 Standing PPO

```powershell
D:\il\env\Scripts\python.exe -B model\step6_train\phase1_stand\train_stand.py `
  --terrain T0 `
  --num_envs 2 `
  --max_iterations 5000 `
  --save_interval 25 `
  --run_name T0_stand_stable
```

Training writes checkpoints under:

```text
logs/rsl_rl/g1_stand/<timestamp>/
```

The newest checkpoint is also copied to `checkpoint_stand.pt` when training
finishes.

### Play Back a Trained Standing Policy

```powershell
D:\il\env\Scripts\python.exe -B model\step6_train\phase1_stand\play_stand.py `
  --terrain T0 `
  --num_envs 1 `
  --load_run 2026-05-09_13-40-12_T0_stand_stable `
  --checkpoint checkpoint_stand.pt
```

For headless validation:

```powershell
D:\il\env\Scripts\python.exe -B model\step6_train\phase1_stand\play_stand.py `
  --headless `
  --terrain T0 `
  --num_envs 1 `
  --max_steps 500 `
  --report_interval 100 `
  --load_run 2026-05-09_13-40-12_T0_stand_stable `
  --checkpoint checkpoint_stand.pt
```

If validation detects a reset/failure, the script prints the exact line:

```text
error
```

## Step 6 AHC Training Framework

Step 6 implements the AHC (Adaptive Humanoid Control) training framework with
terrain system and Phase 1 standing PPO workflow.

### Terrain System (T0-T5)

| Key | Type | Parameters |
|-----|------|------------|
| T0 | Flat plane | — |
| T1 | 10° slope | slope=0.175 rad |
| T2 | 20° slope | slope=0.349 rad |
| T3 | 30° slope | slope=0.524 rad |
| T4 | Stairs | step_height 0.05-0.15m, width 0.3m |
| T5 | Irregular wave | amplitude 0.02-0.08m |

### Phase 1 Standing PPO — Reward Design (24 Terms)

The standing policy uses 24 reward terms organized into 8 categories:

**Base Stability (5 terms):**
| Reward | Weight | Purpose |
|--------|--------|---------|
| `is_alive` | +5.0 | Survival bonus per step |
| `flat_orientation_l2` | -2.0 | Torso tilt penalty (projected gravity deviation) |
| `base_height_l2` | -1.0 | Height deviation from 0.78m target |
| `lin_vel_xy_l2` | -1.5 | Horizontal velocity penalty (standing = zero) |
| `ang_vel_xy_l2` | -1.5 | Angular velocity penalty |

**Energy & Smoothness (3 terms):**
| Reward | Weight | Purpose |
|--------|--------|---------|
| `joint_torques_l2` | -2.5e-5 | Energy efficiency (tiny weight, regularization) |
| `joint_vel_l2` | -5.0e-3 | Joint velocity smoothing |
| `joint_acc_l2` | -5.0e-7 | Joint acceleration smoothing (tiny weight) |

**Joint Safety (4 terms):**
| Reward | Weight | Purpose |
|--------|--------|---------|
| `joint_pos_soft_limits_l2` | -12.0 | Penalty when joint pos exceeds 60% of USD range |
| `joint_pos_hard_limits_l1` | -50.0 | Linear penalty for hard USD position limit excess |
| `joint_vel_usd_limits_l1` | -8.0 | Penalty when joint vel exceeds 30% of USD velocity limit |
| `joint_limit_violation_penalty` | -500.0 | Terminal penalty for any hard limit violation |

**Foot Contact Quality (3 terms, via expanded contact sensors):**
| Reward | Weight | Purpose |
|--------|--------|---------|
| `feet_slide` | -0.5 | Penalize foot sliding while in ground contact |
| `feet_contact_presence` | -1.0 | Penalize missing foot contacts (encourage both feet down) |
| `feet_contact_balance` | -0.5 | Penalize left/right contact force imbalance |

Contact sensors cover: ankle_roll_link, knee_link, elbow_link, wrist_yaw_link, rubber_hand.

**Arm & Posture (6 terms):**
| Reward | Weight | Purpose |
|--------|--------|---------|
| `arm_swing_l2` | -3.0 | Penalize arm swing energy (keep arms still) |
| `arm_swing_asymmetry_l2` | -4.0 | Penalize left/right arm imbalance |
| `joint_deviation_waist` | -0.2 | Waist joints deviation from default pose |
| `joint_deviation_shoulders` | -2.0 | Shoulder pitch/roll/yaw deviation from default pose |
| `joint_deviation_arms` | -1.0 | Arm joints deviation from default pose |
| `joint_symmetry_arms` | -0.2 | Left/right arm symmetry (7 joints per side) |

**Action (1 term):**
| Reward | Weight | Purpose |
|--------|--------|---------|
| `action_rate_l2` | -0.05 | Action change rate penalty |

**Terminal (2 terms):**
| Reward | Weight | Purpose |
|--------|--------|---------|
| `termination_penalty` | -200.0 | Fall penalty (root height < 0.45m) |
| `collapse_ground_contact_penalty` | -500.0 | Multi-limb collapse penalty (3+ limbs ground contact) |

Design principles:
- Hip/knee/ankle joints are **not** posture-constrained — the policy can adapt across terrain
- Waist, shoulders, and arms are constrained to natural standing pose
- Contact sensor rewards (feet_slide, presence, balance) enforce stable ground contact
- Joint safety uses a layered approach: soft limits (60%) → hard limits → termination
- Action targets are clamped to USD joint limits via `ClampedJointPositionAction` (soft limits, 0.005 margin)

### Termination Conditions

| Condition | Threshold |
|-----------|-----------|
| `time_out` | 60 seconds (1200 steps @ 50Hz) |
| `root_height_below_minimum` | < 0.45m (full fall) |
| `joint_pos_out_of_limit` | Hard USD position limit exceeded (0.005 tolerance) |
| `joint_vel_out_of_limit` | Hard USD velocity limit exceeded |
| `collapse_ground_contact` | 3+ limb groups touching ground |

### Key Hyperparameters

| Parameter | Value |
|-----------|-------|
| Action scale | 0.10 |
| Action limit kind | soft (60% of USD range) |
| Action limit margin | 0.005 |
| Episode length | 60s |
| Foot/ground friction | static=2.0, dynamic=2.0 |
| Env spacing | 4.0 |
| Policy frequency | 50Hz (dt=0.005s, decimation=4) |
| Observation | 93-dim (base_ang_vel + projected_gravity + joint_pos_rel + joint_vel_rel + last_action) |
| Action | 29-dim joint position targets (clamped to USD limits) |
| PPO Actor/Critic | [512, 256, 128], ELU activation |
| PPO learning rate | 1e-3 (adaptive) |
